Ethical Hacking Task 02: Network Scanning & Service Enumeration

Target System: localhost (127.0.0.1)

Objective

The objective of this task is to learn the scanning and enumeration phase of ethical hacking using Nmap. This includes identifying open ports, running services, service versions, and operating system information on an authorized system.

Commands Used

1. "nmap --version"
2. "nmap localhost"
3. "nmap -sV localhost"
4. "nmap -O localhost"

Part F – Scan Analysis

1. Which services are currently running?
The scan found services such as Microsoft RPC and SMB running on my computer.

2. Are all open ports necessary?
Not always. Some ports are required for system functions while others should be disabled if unused.

3. Which services could become security risks if misconfigured?
SMB, RDP, FTP and Telnet can become security risks if they are not configured securely.

4. Which port would you disable if it wasn't required?
If Remote Desktop (3389) is not required, I would disable it to reduce attack surface.

Observations

- The localhost scan identified the open ports available on the system.
- Service version detection identified the software running on those ports.
- Operating system detection helped identify the host operating system.
- The scan demonstrated how Nmap is used during the information-gathering phase of ethical hacking.


Recommendations

- Close unnecessary ports.
- Disable unused services.
- Keep software updated.
- Use strong passwords.
- Enable the firewall.
- Perform regular security scans.


Conclusion

During this task, I learned the importance of network scanning and service enumeration in ethical hacking. Using Nmap, I successfully scanned my local machine to identify open ports, running services, and operating system details. I also understood how service version detection helps identify software versions that may contain vulnerabilities. Researching common network ports improved my understanding of the services they provide and the security risks associated with them. I learned that open ports should be monitored regularly and unnecessary services should be disabled to reduce the attack surface. Overall, this task provided practical experience with one of the most important phases of ethical hacking and improved my understanding of network security, system analysis, and safe, authorized scanning techniques.
